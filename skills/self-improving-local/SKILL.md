# Self Improving Local

When a useful lesson appears:

1. Decide whether it is temporary or durable.
2. Store temporary/session details in daily memory notes.
3. Store durable workflow lessons in the most relevant local file:
   - `AGENTS.md` for operating rules
   - `TOOLS.md` for environment-specific notes
   - `MEMORY.md` for long-term user/project memory
   - a skill file if the lesson improves a repeatable skill workflow
4. Keep entries compact and actionable.
5. Prefer updating the exact file future-you will actually read.

See `references/rules.md` for logging rules.
