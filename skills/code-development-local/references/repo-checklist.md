# Repo checklist

## Before edits
1. Identify repo root.
2. Check `git status --short`.
3. Inspect relevant files.
4. Confirm whether tests/build steps are available.

## During work
1. Make focused edits.
2. Re-read changed files.
3. Run the smallest useful verification.
4. Capture durable notes if the repo has non-obvious conventions.

## After work
1. Summarize what changed.
2. Commit in small, understandable units.
3. Note blockers such as missing auth, secrets, runtimes, or upstream access.
